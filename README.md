        # kafka — Что такое топик

        Homework-шаблон для урока **l1_topic_basics** (Что такое топик) на платформе Vibe Learn.

        ## Что делать

        Реализуй idempotent Kafka producer на Go. Шлёт N сообщений в топик `orders`. Требования:
- enable.idempotence=true
- ретраит на network error до 3 раз с экспоненциальным backoff
- корректно закрывает соединение на SIGINT
- логирует partition+offset каждого ack
Тесты в template репо проверят: правильность вызова Flush, поведение под симулированной сетевой ошибкой, graceful shutdown.

## Контекст (из transfer-задачи урока)

У тебя три микросервиса: `orders-api` (REST API, принимает заказы), `inventory` (списывает
остатки), `notifications` (шлёт email). Сейчас `orders-api` синхронно вызывает оба
других сервиса HTTP-запросом — и это уже даёт сбои (notifications упал → orders-api
падает с 5xx). Тебе нужно перевести коммуникацию на Kafka: `orders-api` публикует
событие `OrderCreated` в топик, остальные подписаны и читают.

**Вопрос:** что произойдёт с этим топиком в трёх сценариях:
(a) inventory временно недоступен 10 минут (рестарт пода);
(b) notifications читает топик медленнее, чем туда пишут (lag растёт);
(c) ты хочешь добавить четвёртый микросервис `analytics`, который тоже хочет
    обрабатывать `OrderCreated` (включая исторические события).

## Recap из урока

- Топик — это **именованный поток сообщений**: append-only log, не очередь. Сообщение остаётся после прочтения до retention.ms.
- Партиции дают **горизонтальное масштабирование**: один топик делится на N независимых логов, которые могут жить на разных брокерах и читаются параллельно консьюмерами в группе.
- Брокеры реплицируют партиции (replication.factor по умолчанию 3): один leader + (N-1) followers. Failover автоматический — отсюда «распределённая».
- Ключ сообщения детерминирует партицию (hash(key) % N). Один key → одна партиция → сохранение порядка для этого key без глобальной блокировки.
- Топик ≠ очередь. Топик ≈ Netflix-сериал: эпизоды (партиции) можно перематывать, разные зрители (consumers) могут смотреть параллельно с разных моментов.

        ## Как работать

        1. Платформа Vibe Learn создаёт копию этого репо в твоём GitHub-аккаунте по клику «Начать домашку» на странице урока (через GitHub `/generate`, codecrafters-pattern).
        2. Склонируй копию локально, реализуй TODO в `main.go`, прогони тесты, запушь.
        3. CI (`.github/workflows/ci.yml`) запускает `go vet` + `go test ./...` на каждый push. Платформа слушает результат через webhook от GitHub Actions и обновляет статус домашки на странице урока.

        ## Локальное окружение

        - Go 1.22+
        - Docker + docker-compose — `docker compose -f docker-compose.yml up -d` поднимает 3-нодовый Kafka cluster на портах 9092/9093/9094, использовать в тестах через bootstrap `localhost:9092,localhost:9093,localhost:9094`.

        ## Запуск

        ```bash
        # Поднять локальный Kafka
        docker compose up -d

        # Прогнать тесты (часть из них стартует свой ephemeral testcontainers cluster, часть использует docker-compose выше)
        go test ./...

        # Запустить main (печатает marker; замени stub на реализацию)
        go run .
        ```
